1. مقدمة ونطاق النظام

Hakimo ERP نظام ويب متعدد المستأجرين (Multi-tenant) لإدارة المصنع: كل شركة لها مسار URL خاص /t/{slug}/ وبيانات معزولة في Firestore.

هذا الدليل يغطي:

  • الإنتاج — تقارير يومية، خطط، أوامر شغل، خطوط، مسارات، دورات توريد.
  • المواد التصنيعية — مواد، BOM، تخطيط احتياجات (مرتبط بالإنتاج والمخزون).
  • المخازن — مخازن، أرصدة، حركات، تحويلات، جرد، توجيه مخزون.
  • التكاليف — مراكز تكلفة، تكلفة شهرية، أصول، صحة البيانات.
صفحة /products/raw-materials قديمة (قراءة فقط). استخدم /manufacturing/materials للمواد الجديدة.

2. البنية والتدفقات بين الموديولات

التسلسل التشغيلي الموصى به:

  1. إعداد مخازن + توجيه مخزون من الإعدادات.
  2. تعريف مواد تصنيعية وBOM لكل منتج.
  3. إنشاء خطة إنتاج → توليد احتياجات مواد → أمر شغل.
  4. تسجيل تقرير إنتاج (إدخال سريع أو التقارير) → حركات/تحويلات مخزون تلقائية.
  5. اعتماد تحويلات إن كان التوجيه يتطلب موافقة.
  6. نهاية الشهر: مراكز تكلفة → تكلفة إنتاج شهرية.

9. تشغيل يومي وقائمة تحقق

  • ☐ إعداد مخازن + توجيه مخزون قبل أول تقرير إنتاج
  • ☐ مواد تصنيعية + BOM للمنتجات النشطة
  • ☐ خطوط إنتاج + ربط عمال ومشرفين
  • ☐ دورة: خطة → أمر شغل → تقرير → اعتماد تحويل
  • ☐ نهاية الشهر: مراكز تكلفة → تكلفة شهرية → صحة البيانات

جلسة تدريب مقترحة (90 دقيقة)

  1. 15 د — مقدمة ولوحة التحكم
  2. 25 د — إنتاج + BOM
  3. 25 د — مخازن وتوجيه
  4. 15 د — تكاليف
  5. 10 د — صلاحيات وأسئلة

أسئلة شائعة

  • تحويل يدوي vs تلقائي: التقارير تنشئ production_auto؛ التحويل السريع manual.
  • سالب المخزن: يتطلب صلاحية inventory.finishedStock.allowNegativeApprove عند الاعتماد.
  • خط التعبئة: لا يزيد كمية أمر الشغل بنفس منطق خط التجميع.

10. جاهزية التشغيل الكامل

التشغيل «الكامل» يعني: البيانات والإعدادات جاهزة قبل الاعتماد على المخزون التلقائي والتكلفة الشهرية.

مرحلة A — أسبوع الإعداد (إلزامي)

#المهمةالمسار
A13+ مخازن/inventory/warehouses
A3توجيه مخزون V1 (WIP + تم الصنع)/settings
A4مزامنة ترحيل V1/settings
B1–B2مواد تصنيعية + BOM لكل منتج نشط/manufacturing/materials, /products
C1تكلفة شراء لكل مادةتفاصيل المادة
D1خطوط + ربط عمال/مشرفين/lines
E1–E2مراكز تكلفة + إعدادات عمالة/cost-centers, /cost-settings
تحقق آلي: npm run handover:readiness — يقرأ Firestore ويطبع نسبة الجاهزية.

علامات نجاح

  • لا رسالة «توجيه المخازن غير مكتمل» على /inventory.
  • بعد تقرير إنتاج: حركة/تحويل في المخزون.
  • متوسط تكلفة وحدة > 0 على لوحة Admin بعد إقفال شهر (وليس بيانات ناقصة).

فجوات في دورة التشغيل (ليست صفحات ناقصة)

  • احتياجات المواد: توليد يدوي من الخطة — ليس تلقائياً عند الإنشاء.
  • اعتماد التحويلات: إلزامي إذا requireApprovalForAutoTransfers مفعّل.
  • إقفال التكلفة الشهرية: عملية إدارية من /monthly-costs.

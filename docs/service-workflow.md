# مسار عمل صيانة الأجهزة المنزلية (Factory ERP)

## الفكرة للموظف

الموظف يمشي على خطوات بسيطة؛ النظام يسجّل كل شيء في الخلفية (أحداث، حجوزات قطع، موافقات).

## الحالات الرئيسية

| الحالة | معنى تقريب للموظف |
|--------|-------------------|
| `received` | وارد — تم استلام الجهاز |
| `diagnosing` | تشخيص — الفحص الفني |
| `waiting_approval` | انتظار موافقة العميل على التقدير |
| `waiting_parts` | انتظار وصول / حجز قطع الغيار |
| `repairing` | تنفيذ الإصلاح |
| `testing` | اختبار بعد الإصلاح |
| `ready` | جاهز للتسليم للعميل |
| `delivered` | تم التسليم (إغلاق ناجح) |
| `cancelled` | ملغى (سحب، رفض، إلخ) |
| `unrepairable` | غير قابل للإصلاح (إغلاق تقني) |

## متى `waiting_approval` ومتى `waiting_parts`؟

- **`waiting_approval`**: بعد التشخيص لما في تقدير تكلفة ولازم العميل يوافق قبل ما نشتغل أو نطلب قطع بمبلغ واضح.  
  *قرار تشغيلي مصري شائع: ما نتحركش في شراء/حجز قطع غالية من غير موافقة كتابية أو رابط موافقة.*

- **`waiting_parts`**: العميل وافق (أو الموافقة مش مطلوبة)، والشغل متوقف لأن القطعة لسه مش متاحة فعليًا — بنعمل **حجز** على المخزون (ما يخصمش الرصيد الفيزيائي إلا عند الصرف).

## تدفق مقترح

```mermaid
flowchart LR
  received --> diagnosing
  diagnosing --> waiting_approval
  waiting_approval --> waiting_parts
  waiting_parts --> repairing
  repairing --> testing
  testing --> ready
  ready --> delivered
```

## واجهات الاستخدام

- **قائمة الطلبات**: جدول + **لوحة Kanban** (سحب وإفلات) لتحديث الحالة بسرعة.
- **ورشة الفني** (`/repair/jobs/:id/workspace`): صفحة واحدة موبايل أولاً، شريط إجراءات ثابت أسفل الشاشة.
- **لوحة الصيانة**: مؤشرات متأخرات، عبء فنيين، موديلات متكررة، ربحية تقريبية — تحديث دوري (بدون مستمعات ثقيلة على كل الطلبات).

## ملاحظات ترحيل

طلبات قديمة كانت تستخدم `inspection` أو `repair` تُعرَض في الواجهة كـ `diagnosing` و `repairing` تلقائيًا عند القراءة.

# إصلاح مشاركة تقارير الإنتاج عبر واتساب

تاريخ التعديل: يونيو 2026

## المشكلة التي أبلغ عنها المستخدم

عند مشاركة صورة تقرير الإنتاج عبر واتساب كان السلوك غير ثابت:

| السلوك | الوصف |
|--------|--------|
| تحميل طويل | يظهر «جاري تجهيز الصورة...» لفترة بدون نتيجة واضحة |
| نص فقط | يفتح واتساب مباشرة مع **كابشن (نص)** بدون صورة |
| نجاح كامل | تظهر قائمة المشاركة أو يفتح واتساب **بالصورة** (أحياناً مع النص) |

الصفحات المتأثرة بشكل مباشر:

- **تقارير الإنتاج** (`modules/production/pages/Reports.tsx`) — زر المشاركة في كل صف
- **الإجراء السريع** (`modules/production/pages/QuickAction.tsx`) — زر «مشاركة عبر WhatsApp»

---

## السبب الجذري

### 1) مسار احتياطي خاطئ (السبب الرئيسي لـ «نص بدون صورة»)

الكود القديم في `Reports.tsx` و `QuickAction.tsx` كان يعمل كالتالي:

```
تحويل التقرير → صورة PNG
       ↓
محاولة navigator.share({ files: [صورة] })
       ↓ (فشل أو canShare = false)
تحميل الصورة + فتح whatsapp://send?text=الكابشن
```

المشكلة: الرابط `whatsapp://send?text=...` يفتح محادثة واتساب **بنص مُسبق التعبئة فقط** — لا يرفق الصورة تلقائياً. المستخدم يرى كابشن بدون صورة.

### 2) فحص `canShare` صارم جداً على أندرويد

على Chrome للأندرويد، `navigator.canShare({ files: [...] })` أحياناً يرجع `false` رغم أن `navigator.share()` ينجح فعلياً.

الكود القديم كان:

```ts
if (!navigator.canShare?.({ files: [shareFile] })) return null;
```

أي فشل مبكر → انتقال فوري للمسار الاحتياطي (نص فقط).

### 3) تأخر إظهار حالة التحميل

في صفحة التقارير، حالة `sharingReportId` (التي تُظهر «جاري تجهيز الصورة...») كانت تُفعَّل **بعد** انتظار رسم البطاقة، فيبدو للمستخدم أن الزر لا يستجيب ثم يتوقف أو يفتح واتساب بشكل غير متوقع.

### 4) عدم وجود مهلة زمنية لتحويل الصورة

دالة `exportNodeToPng` قد تستغرق وقتاً (خطوط، صور، انتظار رسم). بدون timeout قد يبقى الزر في حالة تحميل لفترة طويلة.

## الحل المُطبَّق

### دالة موحّدة: `shareImageBlobToWhatsApp`

تم استخراج منطق المشاركة إلى دالة واحدة في:

`utils/reportExport.ts`

وتستخدمها الآن:

- `shareToWhatsApp` (التقاط DOM ثم مشاركة)
- `Reports.tsx` (بعد `exportNodeToPng`)
- `QuickAction.tsx` (بعد `exportNodeToPng`)

### تدفق المشاركة الجديد

```
تحويل التقرير → Blob صورة
       ↓
shareImageBlobToWhatsApp(blob, عنوان, { caption })
       ↓
┌─ على الموبايل ─────────────────────────────────────┐
│ 1) مشاركة PNG (ملف + نص إن وُجد)                  │
│ 2) إن فشل: مشاركة JPEG بجودة 0.97                  │
│ 3) إن فشل: مشاركة ملف فقط + نسخ النص للحافظة     │
└────────────────────────────────────────────────────┘
       ↓ (فشل كامل)
تحميل الصورة على الجهاز
نسخ الكابشن للحافظة (إن وُجد)
فتح whatsapp://send  ← بدون ?text=  (محادثة فارغة لإرفاق الصورة يدوياً)
```

على **سطح المكتب**: تحميل + نسخ للحافظة إن أمكن + فتح `web.whatsapp.com`.

### الفروقات عن السلوك القديم

| قبل | بعد |
|-----|-----|
| فشل `canShare` → لا محاولة مشاركة | على الموبايل: محاولة `share()` حتى لو `canShare` = false |
| احتياطي: `whatsapp://send?text=كابشن` | احتياطي: تحميل صورة + `whatsapp://send` بدون نص |
| منطق مكرر في ملفين | دالة واحدة `shareImageBlobToWhatsApp` |
| تحميل يظهر متأخراً | `sharingReportId` / `sharingImage` من أول لحظة |
| لا timeout | مهلة 20 ثانية لتحويل الصورة |

## رسائل المستخدم بعد المشاركة

تُعرَض عبر `getShareResultFeedbackMessage` في `reportExport.ts`:

| الحالة | الرسالة (مختصر) |
|--------|------------------|
| مشاركة ناجحة مع نص منسوخ | «تم إرسال الصورة. تم نسخ التفاصيل — الصقها كتعليق...» |
| تحميل + نسخ | «تم تحميل الصورة ونسخ التفاصيل — أرفق الصورة من المجلد ثم الصق النص» |
| تحميل فقط | «تم تحميل صورة التقرير — أرفقها في محادثة واتساب» |
| إلغاء المستخدم | لا رسالة |

## كيفية الاختبار

### على الموبايل (أندرويد / iOS)

1. افتح **تقارير الإنتاج** أو **الإجراء السريع**.
2. اضغط مشاركة واتساب لتقرير واحد.
3. **الحالة المثالية:** تظهر قائمة المشاركة → اختر واتساب → الصورة (والنص إن دعمه النظام).
4. **الاحتياطي:** تُحمَّل الصورة، يُنسخ النص، يفتح واتساب — أرفق الصورة من المجلد والصق التعليق.

### على الكمبيوتر

1. نفس الخطوات.
2. إن لم تدعم المتصفح مشاركة الملفات: تُحمَّل الصورة ويُفتح WhatsApp Web.

### حالات يجب التحقق منها

- [ ] تقرير منتج نهائي واحد
- [ ] تقرير تعبئة متعدد المنتجات
- [ ] إلغاء قائمة المشاركة (لا يجب أن تظهر رسالة خطأ)
- [ ] الضغط السريع مرتين (يجب أن يُتجاهل الطلب الثاني — `shareWhatsAppLockRef`)

## ملاحظات تقنية

### تحويل التقرير لصورة

ما زال يستخدم `exportNodeToPng` من `src/shared/utils/exportNodeToImage.ts` (مكتبة `html-to-image`) بعرض ثابت **1080px** للبطاقة المخفية `ProductionReportShareCard`.

المشاركة الجماعية من صفحة التقارير (كل التقارير دفعة واحدة) تستخدم `shareToWhatsApp` مع `html2canvas` — مسار منفصل لكن نفس دالة المشاركة `shareImageBlobToWhatsApp` بعد التقاط الصورة.

### قيود المتصفح (لا يمكن تجاوزها بالكود)

- iOS Safari قد لا يدعم `files + text` في نفس استدعاء `share`.
- واتساب لا يقبل إرفاق صورة عبر deep link — الاحتياطي دائماً يتطلب إرفاق يدوي عند فشل Web Share API.
- حجم الصورة الكبير قد يبطئ التحويل أو يرفضه بعض الأجهزة.
